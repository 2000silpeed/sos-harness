# Six-Space Review — 6-공간 코드 리뷰

코드를 6개 독립적 분석 차원으로 검토한다. 일반 코드 리뷰가 놓치는 구조적/시간적/인과적 문제를 발견한다.

## 6개 분석 공간

### 1. 계층 공간 (Hierarchy)
**핵심 질문**: "책임이 올바른 수준에 있는가?"
- 컴포넌트/모듈이 올바른 계층에 위치하는가?
- 상위 모듈이 하위 구현에 의존하지 않는가? (의존성 역전)
- 추상화 수준이 일관적인가?
- **경고**: 수준 간 결론 이동 오류 (상위 계층의 결론을 하위에 무비판적으로 전달)

### 2. 시간 공간 (Temporal)
**핵심 질문**: "시간 순서가 올바른가?"
- 상태 초기화/정리 타이밍은 적절한가?
- 레이스 컨디션 가능성은?
- 이벤트 순서 보장이 필요한 곳에서 보장되는가?
- 타임아웃/재시도 전략은?
- **패턴**: 생명주기 분석 (생성→활성→비활성→소멸)

### 3. 재귀 공간 (Recursive)
**핵심 질문**: "재사용 가능한 패턴인가?"
- 3회 이상 반복되는 패턴이 추상화되었는가?
- 단, 조기 추상화는 아닌가? (3번 반복 전까지는 복붙 OK)
- 컴포넌트 합성이 가능한 구조인가?
- **경고**: 프랙탈 자기유사성 위반 (부분과 전체의 구조가 불일치)

### 4. 구조 공간 (Structural)
**핵심 질문**: "의존성이 건강한가?"
- 순환 참조 없는가?
- 모듈 간 결합도가 적절한가?
- 변경 시 영향 범위가 제한적인가?
- 도메인 모델(온톨로지)과 코드 구조가 일치하는가?
- **도구**: `import` 분석, 디렉토리 구조 검토

### 5. 인과 공간 (Causal)
**핵심 질문**: "실패하면 어떻게 되는가?"
- 에러 처리가 적절한가?
- 에러 A → 연쇄 반응 B → C의 전파 경로는?
- 복구 가능한가? 롤백 전략은?
- 데이터 정합성이 보장되는가?
- **패턴**: Impact Analysis ("이것을 바꾸면 무엇이 영향받는가?")

### 6. 통합 공간 (Cross-space)
**핵심 질문**: "전체적으로 일관성 있는가?"
- 위 5개 공간의 분석을 종합
- 도메인 모델의 클래스/관계/공리가 코드에 충실히 반영되었는가?
- 기술 부채(tech debt) 식별
- 총평 및 개선 우선순위
